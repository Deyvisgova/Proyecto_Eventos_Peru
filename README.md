# Proyecto Eventos Perú
Incluye el backend (Spring Boot) y el frontend (Angular) en un solo repositorio.

## Diagramas clave
Los siguientes recursos visuales ayudan a explicar conceptos clave de la solución:

- **Arquitectura por capas**: visión renovada con flechas, colores e iconos que separa presentación, aplicación, dominio e infraestructura. ![Arquitectura por capas](docs/imagenes/arquitectura_por_capas.svg)
- **Modelo de procesos (BPMN)**: flujo de compra de entradas entre cliente, API y pasarela de pago. ![BPMN](docs/imagenes/modelo_procesos_bpmn.svg)
- **Diagrama de clases**: entidades principales y operaciones del dominio. ![Diagrama de clases](docs/imagenes/diagrama_clases.svg)
- **Modelo de datos (ER)**: llaves primarias/foráneas y relaciones. ![Modelo de datos](docs/imagenes/modelo_datos.svg)
- **Implementación de patrones**: muestra cómo Factory Method, Strategy y Observer/Mediator colaboran en los servicios de eventos. ![Implementación de patrones](docs/imagenes/implementacion_patrones.svg)

Para ver las imágenes directamente y en mayor tamaño dentro del repositorio, consulta `docs/imagenes/README.md`.
